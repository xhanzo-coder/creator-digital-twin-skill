# 模式 B：资产与记忆管理 (Asset & Memory Co-creation)

**定位**：数字分身的”外挂大脑”。不再是被动记录，而是通过”灵感对撞”将碎片点子转化为结构化资产，**并从用户纠正中学习偏好**。

---

## ⛔ 纠正学习优先触发 (Priority: 0)

**当用户说以下话时，立即进入纠正学习模式，跳过其他所有流程：**

| 触发词 | 解析 |
|-------|------|
| “这不对” / “不对” / “不是这样” | 用户否定了生成内容 |
| “我改一下” / “让我改改” | 用户要自己修改 |
| “这里写错了” / “这不是我的风格” | 具体指出问题 |
| 用户直接提供修改后版本 | 已完成修改，需要 Diff 分析 |

### 纠正学习执行流程

```
用户说”这不对”
    ↓
立即停止当前任务
    ↓
追问具体问题：”请问哪些地方需要修改？”
    ↓
用户提供修改内容 或 直接提供修改版本
    ↓
执行 Step 5: Diff 分析 + 偏好捕捉
    ↓
展示学习报告，确认后写入档案
```

**⚠️ 纠正学习不能跳过**：当用户表示不满意时，必须执行偏好捕捉，不能直接重写。

## Step 1: 灵感接收与全库扫描 (Multi-dimensional Scanning)

当你提供一个想法、点子、金句或一段经历时，系统立即启动**背景关联检查**：

### 1. 维度 A：历史内容关联 (Contextual Search)
- **扫描路径**：`./.creator-space/content/published/`
- **逻辑**：寻找相似的主题或观点，提醒你：“你曾在 [日期] 写过类似内容，是延续还是更新？”

### 2. 维度 B：冷启动策略关联 (Cold Start Strategy)
- **扫描路径**：`./.creator-space/assets/strategies/` 或 `./.creator-space/system/profile.json`
- **逻辑**：若无历史文章，则调用你学习过的方法论（如 `dontbesilent` 框架）或你的人设边界（`stance_topics.json`）。
- **反馈示例**： “这个想法非常符合你主张的‘AI 提效’人设。按照你存过的‘爆款文稿库’结构，这个点子可以作为‘反直觉切入点’。”

### 3. 维度 C：实时情报对冲 (News Pulse)
- **扫描路径**：`./.creator-space/news_sources/`
- **逻辑**：检索最近 7 天的爆款新闻。
- **反馈示例**： “你这个关于‘数字员工’的想法，正好呼应了昨天那条关于 GPT-5 Agent 能力的新闻，建议结合起来做一次‘深度点评’。”

---

## Step 2: 探讨式共创 (Co-creation Dialogue)

系统不应只回复“已记录”，而应提供 **2-3 个深化建议**：
1. **角度拆解**：这个点子可以发小红书（教程向）还是 X（观点向）？
2. **素材补全**：询问用户是否需要调用已有的“金句库”或“案例库”来填充这个点子。
3. **任务转化**：询问“是否直接生成初步大纲？”

---

## Step 3: 隐式记录与资产固化 (Implicit Capture)

在探讨过程中，系统自动识别并格式化以下资产：

| 资产类型 | 存放路径 | 判定标准 |
| :--- | :--- | :--- |
| **点子 (Ideas)** | `assets/ideas/` | 尚未成熟的选题、灵感片段、未来计划 |
| **概念 (Concepts)** | `assets/concepts/` | 你独特的定义、方法论名词（如“生产型兴趣”） |
| **金句 (Quotes)** | `assets/quotes/` | 具有穿透力的短句、对话中蹦出的精彩表达 |
| **案例 (Cases)** | `assets/cases/` | 具体的例子、调研数据、成功/失败的事实 |
| **经历 (Stories)** | `memory/stories.jsonl` | 你的个人故事、情绪、具体发生的事件 |
| **观点 (Beliefs)** | `memory/beliefs.json` | 价值观的声明、对某事的深度看法 |

**👉 确认逻辑**：对话结束前，系统汇总准备记录的项目，用户回复“确认”或“好”后正式写入 JSONL。

---

## Step 4: 更新索引与关联 (Indexing)

写入后，自动更新 `assets/index.json`。
如果该点子关联了某条新闻，在 `metadata` 中记录 `news_hash`，实现“情报与灵感”的强绑定。

---

## Step 5: 实时修改捕捉 (Real-time Edit Capture) - **[整合模式 F]**

**触发场景**：用户提供 AI 生成的稿件及其修改后的版本（或直接发回修改后的稿件）。

### ⚠️ 这是学习用户偏好的关键时刻

**必须执行**：
1. **暂停一切其他任务**
2. **执行完整的 Diff 分析**
3. **向用户展示学到了什么**
4. **获得用户确认后才写入档案**

### 执行流程

**Step 5.1: Diff 对比分析**

对比 AI 原稿 vs 用户修改稿，提取以下维度：

```markdown
## Diff 分析报告

### 词汇偏好变化
| AI 原词 | 用户替换 | 偏好类型 |
|--------|---------|---------|
| [被删词] | [替换词] | [更专业/更口语/更精准] |
| ... | ... | ... |

### 语气变化
- 原：[AI 的语气描述]
- 改：[用户的语气描述]
- 结论：用户偏好 [更直接/更委婉/更专业/更轻松] 的语气

### 句式偏好
- 原：[AI 的句式特点]
- 改：[用户的句式特点]
- 结论：用户偏好 [短句/长句/对话式/陈述式]

### 内容修正
- 用户删除了什么（不想要的内容）
- 用户新增了什么（重视的内容）
- 用户调整了什么（表达方式）

### 事实核对
- 用户纠正了哪些事实错误？
- 这说明用户对 [某领域] 的态度是 [严谨/轻松/不在乎]
```

**Step 5.2: 更新偏好档案**

将学到的偏好写入以下文件：

| 文件 | 内容 |
|------|------|
| `persona/voice_style.json` | 语气、节奏、词汇偏好 |
| `persona/do_dont_say.json` | **do**: 用户喜欢的表达 / **dont**: 用户讨厌的表达 |
| `persona/tone_by_scene.json` | 不同场景的语气调整 |
| `memory/beliefs.json` | 用户表达的价值观 |

**Step 5.3: 展示学习报告并确认**

```
## 📝 我学到了这些偏好

1. **词汇层面**：你不喜欢”AI要变天了”这种夸张表达，偏好更严谨的表述
2. **语气层面**：你偏好专业但不装逼的风格，反感情绪化宣泄
3. **内容层面**：你是AI工具早期用户，不是”把AI当问答机”的新手
4. **事实层面**：你对模型版本信息很敏感，Claude最新是4.6

这些理解正确吗？我会记住这些偏好应用到后续创作中。
```

**Step 5.4: 用户确认后写入**

用户回复”对”、”是的”、”确认”后正式写入 JSON 文件。

## Step 6: 偏好捕捉示例 (Examples)

### 示例 1：用户纠正事实

**用户输入**：
> “Claude 最新是 4.6，不是 4。然后 GPT-5.4 是这篇的主角，不应该把它放进'让我疲惫的列表'”

**系统学习**：
```json
// 写入 persona/do_dont_say.json
{
  “do”: [“确保模型版本准确”, “主角不要和其他角色并列”],
  “dont”: [“编造模型版本”, “把主角放进负面列表”]
}

// 写入 memory/beliefs.json
{
  “AI工具使用”: {
    “stance”: “早期用户，不是新手”,
    “accuracy”: “对版本信息很敏感，要求准确”
  }
}
```

### 示例 2：用户纠正风格

**用户输入**：
> “这不是我的日常。我很早就用AI编程和写作了，不会问'帮我写个函数'这种问题”

**系统学习**：
```json
// 写入 persona/voice_style.json
{
  “user_level”: “advanced”,
  “background”: “AI工具早期用户，编程+写作”,
  “preferences”: {
    “self_description”: “不会用新手式的表达”
  }
}

// 写入 persona/do_dont_say.json
{
  “dont”: [
    “把用户描述成AI新手”,
    “写'帮我写个函数'这种新手对话”,
    “写'把AI当问答机'这种不符合用户实际情况的描述”
  ]
}
```

### 示例 3：用户纠正标题

**用户输入**：
> “标题太消极了，不符合我的思维模式”

**系统学习**：
```json
// 写入 persona/do_dont_say.json
{
  “dont”: [“消极情绪标题”, “制造焦虑的标题”],
  “do”: [“积极/中立标题”, “制造好奇而非焦虑”]
}

// 写入 memory/beliefs.json
{
  “思维模式”: {
    “倾向”: “积极/中立，不喜欢消极表达”
  }
}
```
