# 模式 G：外部材料学习模式 (External Learning)

**触发条件**：
- 用户说"学习这个方法论"、"帮我提炼这篇文章"
- 用户提供外部链接、文章、书籍摘录

**核心目标**：提炼优质内容的写作技巧，选择性融入用户风格

## Step 1: 内容获取与识别

```bash
# 若是链接，使用 WebFetch 获取
WebFetch(url)

# 若是本地文件，使用 Read
Read(file_path)
```

**询问用户**：
```
📌 请问这是：
1. 你自己的文章（将直接学习你的风格）
2. 别人的优质内容（将提炼方法论供你选择）

请选择 1 或 2：
```

## Step 2: 内容提炼

### 若选择 1（自己的）

- 跳转到 **模式 E：风格学习**

### 若选择 2（别人的）

**提炼维度**：

1. **写作技巧**：
   - 标题公式（如：数字+痛点+解决方案）
   - 开头方法（如：场景化开头、反常识开头）
   - 论证结构（如：SCQA结构、黄金圈法则）

2. **表达技巧**：
   - 比喻手法（如：将抽象概念比作具体事物）
   - 金句公式（如：A不是B，而是C）
   - 情绪调动（如：反问、共鸣、制造悬念）

3. **内容策略**：
- 选题方向（如：痛点型、方法论型、案例型）
- 价值主张（如：节省时间、降低焦虑、提升效率）

### Step 2.5: 兼容性校验（v4 新增）

提炼后先做 2 个评分，再决定是否可学习：

1. **风格兼容评分**（0-100）  
   与 `persona/voice_style.json` 对比：语气、句式、叙事节奏是否接近。
2. **价值观冲突评分**（0-100，分数越高冲突越大）  
   与 `memory/beliefs.json`、`persona/stance_topics.json` 对比：观点是否冲突。

建议阈值：
- 风格兼容 >= 60 且 价值观冲突 <= 30：可直接学习
- 其余情况：仅作为参考，不写入核心人格档案

**提炼示例输出**：
```
📚 内容提炼完成！从《小红书爆款指南》中学习到：

【写作技巧】
✓ 标题公式：数字+情绪词+承诺
  示例："3天涨粉5000！我用这5个技巧"

✓ 开头方法：直接抛出痛点 + 解决方案预告
  示例："还在为涨粉发愁？我找到了答案"

【表达技巧】
✓ 多用疑问句引发共鸣
  示例："你是不是也遇到过这种情况？"

✓ Emoji 使用规律：每50字一个，情绪点必加

【内容策略】
✓ 选题偏好：痛点+解决方案型（80%）
✓ 结构模式：问题→分析→方法→案例

---

💡 哪些技巧值得融入你的风格？（可多选）
1. 标题公式
2. 疑问句引发共鸣
3. Emoji 使用规律
4. SCQA 结构
5. 全部融入
6. 都不需要

请输入编号（如：1,2,3）：
```

## Step 3: 选择性融入

**根据用户选择**，将技巧写入 `system/profile.json` 的 `learned_techniques` 字段；
若涉及价值观层面更新，同时写入 `memory/beliefs.json` 的变更记录。

```json
{
  "learned_techniques": {
    "from_xiaohongshu_guide": {
      "learned_at": "2026-02-06T16:00:00Z",
      "source": "《小红书爆款指南》",
      "techniques": [
        {
          "type": "title_formula",
          "pattern": "数字+情绪词+承诺",
          "example": "3天涨粉5000！我用这5个技巧"
        },
        {
          "type": "emoji_usage",
          "rule": "每50字一个，情绪点必加"
        }
      ]
    }
  }
}
```

## Step 4: 完成反馈

```
✅ 外部材料学习完成！
📚 已从《小红书爆款指南》中提炼 2 项技巧
🔄 风格档案已更新

💡 下次写小红书内容时，系统将：
   - 使用"数字+情绪词+承诺"生成标题
   - 每50字添加一个 Emoji
```
