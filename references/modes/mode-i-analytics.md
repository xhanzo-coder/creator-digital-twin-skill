## Step 1: 发布存证 (Publish Logging)

**触发条件**：”我刚发了...”、”这是发布链接”、”同步一下发布记录”。

**执行流程**：
1. **获取元数据**：记录标题、平台（小红书/公众号/X等）、发布时间、原文链接。
2. **建立关联**：自动关联该内容在 `content/published/` 中的原稿。
3. **初始化指标**：在 `analytics/performance.jsonl` 中创建记录，指标初始设为 0。

**输出格式**：
```json
{
  “id”: “2026-03-06-ai-boundary”,
  “title”: “AI 圈大新闻：这次不是卷参数，是卷\”边界感\””,
  “platform”: “xiaohongshu”,
  “published_at”: “2026-03-06T10:00:00”,
  “url”: “https://xiaohongshu.com/note/xxx”,
  “draft_path”: “.creator-space/content/drafts/2026-03-06-ai-boundary/”,
  “metrics”: {
    “views”: 0,
    “likes”: 0,
    “collects”: 0,
    “comments”: 0,
    “shares”: 0
  },
  “status”: “published”
}
```

## Step 2: 指标更新 (Metrics Update)

**触发条件**：”更新数据”、”上周那篇爆了”、”同步一下阅读量”。

**各平台核心指标对标**：

| 平台 | 必填指标 (KPIs) | 选填指标 |
| :--- | :--- | :--- |
| **小红书** | 阅读、点赞、**收藏** | 评论、转粉 |
| **公众号** | 阅读、**分享**、在看 | 点赞、留言 |
| **X / Twitter** | 展现量 (Views)、点赞 | **转发 (RT)**、回复 |

**执行流程**：
1. **确认条目**：系统列出近期未结案的发布记录供用户选择。
2. **指标录入**：按平台引导用户输入关键数据。
3. **计算权重**：对比历史平均值，判定是否为”爆款（Top 10%）”或”优质（Top 25%）”。

**爆款判定逻辑**：
```markdown
### 爆款判定标准

**小红书**：
- 🔥 爆款：收藏 > 历史平均 × 2 或 收藏率 > 5%
- ✨ 优质：收藏 > 历史平均 × 1.5 或 收藏率 > 3%

**公众号**：
- 🔥 爆款：分享 > 历史平均 × 2 或 分享率 > 2%
- ✨ 优质：分享 > 历史平均 × 1.5 或 分享率 > 1%

**X / Twitter**：
- 🔥 爆款：转发 > 历史平均 × 2 或 转发率 > 1%
- ✨ 优质：转发 > 历史平均 × 1.5 或 转发率 > 0.5%
```

## Step 3: 爆款反向提炼与策略进化 (Evolution)

这是系统进化的核心。当某篇内容被判定为”爆款”时，系统启动**特征溯源分析**：

1. **标题逆向**：分析该标题采用了哪种情绪钩子（数字、悬念、反直觉）。
2. **结构复盘**：分析前 3 句的留存策略（Hook）和正文的节奏感。
3. **策略归档**：
   - 提取成功模式，更新至 `assets/strategies/` 下的对应平台方法论。
   - **更新人设**：若某个观点引发高转发，同步更新 `persona/stance_topics.json` 的权重。

**⚠️ 必须使用 Write 工具更新策略文件**：
- 路径：`assets/strategies/viral_[主题]_[日期].md`
- 格式：
  ```markdown
  # 爆款策略：[主题]

  ## 数据表现
  - 平台：[平台名]
  - 发布日期：[日期]
  - 核心指标：[数据]
  - 爆款类型：[收藏型/分享型/评论型]

  ## 标题分析
  [标题逆向分析结果]

  ## 结构分析
  [结构复盘结果]

  ## 可复用框架
  [提取的可复用框架]
  ```

### 题材分析
