## 路径 1：选题写作流程

### Step 1: 资产自检 (关键前置步骤)

**在联网调研前**，必须先检索本地知识资产库：

```bash
# 示例：用户选题 "AI Agent 的应用场景"
# 检索操作
grep -r "AI Agent" .writing-style/assets/concepts/
grep -r "Agent" .writing-style/assets/quotes/
```

**若发现相关资产**：
- 主动提示："发现你之前定义过 'AI Agent' 概念，是否复用？"
- 展示既有定义/金句，询问是否融入本次写作

### Step 2: 阶梯式调研

**第一阶梯：本地新闻库线索**
- 检索 `news_sources/daily/` 最近 7 天的数据
- 寻找相关案例和最新动态

**第二阶梯：定向补全**
- 针对新闻摘要执行 WebFetch 获取全文
- 补充细节数据点

**第三阶梯：全网穿透搜索**
- 使用 WebSearch，保持用户原词（不擅自修改专有名词）
- 优先查找官方文档、权威报道

**素材颗粒化提取**：
- 提取并标记：[核心数据]、[专家金句]、[技术术语]

### Step 3: 生成写作角度 + 平台适配建议 (融入用户风格)

**🔴 关键步骤：写作前必须先读取用户风格**

```bash
# 读取用户风格档案
profile_initialized=$(cat ./.writing-style/system/profile.json | jq -r '.initialized')

if [ "$profile_initialized" == "true" ]; then
    echo "✅ 已加载你的个人风格，生成的角度将符合你的表达习惯"
    # 读取风格特征
    tone=$(cat ./.writing-style/system/profile.json | jq -r '.expression_preference.tone')
    thinking_mode=$(cat ./.writing-style/system/profile.json | jq -r '.thinking_mode.logic_type')
else
    echo "⚠️ 当前使用默认风格，建议提供历史文章以获得更符合你风格的角度"
fi
```

**生成 3 个角度**，**每个角度必须符合用户的风格特征**：

示例（假设用户风格是：口语化、案例驱动、犀利直接）：
```
📝 写作角度建议（已根据你的风格定制）：

1. 【案例拆解角度】：扒一扒 AI Agent 的 3 个真实翻车案例
   - 符合你的：案例驱动思维、犀利直接的语气
   - 适合平台：小红书（干货+吐槽）

2. 【个人实测角度】：我用 AI Agent 一周，发现了这些坑
   - 符合你的：第一人称表达、口语化风格
   - 适合平台：公众号（深度体验文）

3. 【反常识角度】：别被忽悠了，AI Agent 其实没那么神
   - 符合你的：批判性思维、直接切入正题
   - 适合平台：Twitter Thread（观点型）

请选择角度（输入 1/2/3）：
```

**🛑 强制停顿**：等待用户选择，不得自动继续。

### Step 4: 平台规则注入 + 专项流程

**用户选择角度后**，执行平台专项流程（见下方"平台专项流程"）

## 路径 2：框架写作流程

**适用场景**：用户已提供大纲、要点或框架结构。

### Step 1: 冲突预警检查

```bash
# 读取平台规则
cat .writing-style/platform_rules/xiaohongshu.json

# 检查项目：
# 1. 用户框架字数预估是否超出平台限制？
# 2. 框架风格是否与平台调性冲突？
# 3. 结构是否符合平台最佳实践？
```

**若发现冲突**：
- 提示用户："你的框架预计 2000 字，但小红书建议 800-1200 字，是否调整？"

### Step 2: 精准素材补全

**针对框架弱项**进行定向调研：

```bash
# 示例：用户框架中 "第三部分：案例分析" 缺少具体案例
# 执行定向搜索
WebSearch("AI Agent 实际应用案例 2026")
```

**素材补强优先级**：
1. 检查 `news_sources/daily/` 最近新闻（最新案例）
2. 检查 `knowledge_base/` 既有资产（历史案例）
3. 全网搜索补充（外部案例）

### Step 3: 用户确认后执行

- 展示补强后的框架
- 询问用户是否满意
- 继续执行 **路径 1 的 Step 4**（平台规则注入 + 质量控制）

## 平台专项流程

### 🔴 小红书专项流程

1. **读取规则**：
```bash
cat .writing-style/platform_rules/xiaohongshu.json
```

2. **生成爆款标题**（5个选项）：
```
📌 标题候选（请选择 1 个）：
1. 🔥 AI Agent 太强了！这 5 个场景你必须知道！
2. ⚡ 揭秘！AI Agent 是如何改变工作流的
3. 💡 实测！用 AI Agent 3 天完成 1 个月工作
4. 🚀 AI Agent 应用场景全解析（附工具清单）
5. 🎯 别再手动了！AI Agent 帮你自动化这些任务

请选择标题（输入 1-5）：
```

**🛑 强制停顿**：等待用户选择标题。

3. **封面视觉建议**：
```
📸 封面文字建议：
主标题：AI Agent 实战
副标题：5 个场景 + 工具清单
```

4. **写正文**：
   - 严格执行 JSON 中的 Emoji 频率（每 50-80 字一个）
   - 字数限制（800-1200 字）
   - 话题标签数量（3-5 个）

### 🔵 Twitter/微信专项流程

**Twitter**：
1. 确认 Thread 大纲
2. 拆解为多条推文，第一条必须包含 Hook
3. 每条推文 ≤ 280 字符

**微信公众号**：
1. 深度扩充内容（3000-5000 字）
2. 添加小标题、引言、总结
3. 排版建议（段落间距、重点加粗）
