# 模式 F：实时学习模式 (Real-time Learning)

**触发条件**：
- 用户说"我修改了这篇文章，帮我学习一下"
- 用户提供 AI 原文 + 用户修改后的版本
- 系统检测到用户对 AI 生成的文章不满意

**核心目标**：学习用户的修改习惯，理解"什么样的表达是用户喜欢的"

## Step 1: 对比分析

```bash
# 使用 diff 工具对比原文和修改后的文本
diff -u ai_draft.md user_edited.md > changes.diff
```

**识别修改类型**：
1. **词汇替换**：AI 写"至关重要" → 用户改"很关键"
2. **句式调整**：AI 写长句 → 用户拆成短句
3. **段落重组**：AI 分3段 → 用户合并为2段
4. **内容删减**：AI 写了某段 → 用户删除
5. **内容新增**：AI 未写 → 用户补充

## Step 2: 提取学习要点

**分析每处修改的原因**（基于现有风格档案对比）：

示例输出：
```
📝 修改分析报告：

修改 1:
原文：这项技术具有至关重要的意义
改为：这个技术挺关键的
分析：
  ✓ 词汇偏好：避免"至关重要"，更喜欢"挺+形容词"
  ✓ 口语化：从书面语改为口语表达
  ✓ 亲切感：用"这个"替代"这项"

修改 2:
原文：此外，它还能提升效率。同时，它也降低了成本。
改为：一方面提升效率，另一方面降低成本。
分析：
  ✓ 避免AI词汇："此外"、"同时"
  ✓ 句式偏好：合并为并列句
  ✓ 逻辑连接：喜欢用"一方面...另一方面"

修改 3:
原文：[删除整段关于历史背景的描述]
分析：
  ✓ 内容偏好：不喜欢过多历史铺垫，直接切入正题
```

## Step 3: 更新风格档案

**将学习要点写入 `system/profile.json`**：

```json
{
  "learning_count": 9,
  "last_updated": "2026-02-06T15:30:00Z",

  "learned_from_edits": {
    "vocabulary_corrections": [
      {"avoid": "至关重要", "prefer": "挺关键的"},
      {"avoid": "此外", "prefer": "一方面"},
      {"avoid": "同时", "prefer": "另一方面"}
    ],
    "sentence_preferences": [
      {"avoid": "多个短句连续", "prefer": "合并为并列句"}
    ],
    "content_preferences": [
      {"reduce": "历史背景铺垫", "increase": "直接切入正题"}
    ]
  }
}
```

## Step 4: 反馈与确认

```
✅ 实时学习完成！
📊 本次学习到 3 处风格偏好
🔄 风格档案已更新

💡 下次写作时将避免：
   - 使用"至关重要"、"此外"等AI词汇
   - 过多历史背景铺垫

💡 下次写作时将优先：
   - 使用"挺+形容词"的口语表达
   - 合并短句为并列结构
```
