# 模式 E：风格学习模式 (Style Learning)

**触发条件**：
- 用户明确说"学习我的风格"、"分析这篇文章"
- 用户提供文件夹路径（检测到路径格式）
- 任何时候用户想更新个人风格档案

## Step 1: 识别输入类型

```bash
# 判断用户输入
if [[ $input =~ ^[A-Za-z]:\\ ]] || [[ $input =~ ^/ ]]; then
    # 检测到路径格式
    MODE="batch_learning"
else
    # 单篇文章学习
    MODE="single_learning"
fi
```

## Step 2: 文章分析与风格提取

### 批量学习模式（推荐3-5篇文章）

1. 读取文件夹中的所有文章
2. 对每篇文章执行风格分析（参考首次学习的阶段2）
3. 聚合分析结果，提取共性特征

### 单篇学习模式

1. 分析单篇文章的风格特征
2. 与现有 `system/profile.json` 进行对比
3. 识别新特征和差异点

## Step 3: 更新风格档案

### 若 `initialized: false`（首次学习）

- 执行完整的风格提取
- 生成新的 `profile.json`（参考首次设置指南的阶段3）
- 设置 `initialized: true`

### 若 `initialized: true`（增量学习）

- 对比新文章与现有风格的差异
- 展示发现的新特征：

```
📊 风格分析完成！发现以下新特征：
✨ 新增高频词：["其实"、"说白了"]
✨ 疑问句使用频率：从 0.3 增加到 0.5
✨ 新发现的表达习惯：喜欢用"你想想看"引导读者思考

💡 是否将这些新特征融入你的风格档案？(是/否/部分融入)
```

- 根据用户选择更新 `system/profile.json`
- 增加 `learning_count` 计数
- 更新 `last_updated` 时间戳

## Step 4: 完成反馈

```
✅ 风格学习完成！
📊 已学习文章数：累计 8 篇
🔄 风格档案已更新：./.writing-style/system/profile.json
💡 下次写作将应用最新的风格特征
```
