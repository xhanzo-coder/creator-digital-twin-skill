## Step 0: 日期锚点解析 (Date Anchor Resolution)

**在展示新闻前，必须先解析用户意图中的日期范围：**

| 用户输入 | 日期范围 | 数据文件 |
|---------|---------|---------|
| "今天有什么新闻" | 当天 | `daily/YYYY-MM-DD.json`（当天） |
| "昨天有什么新闻" | 前一天 | `daily/YYYY-MM-DD.json`（前一天） |
| "最近有什么新闻" | 最近 3 天 | 合并多个 daily 文件 |
| "X月X日的新闻" | 指定日期 | `daily/YYYY-MM-DD.json` |
| "过去有什么可写的" | 最近 30 天（含已读） | 合并多个 daily 文件，**包含已读** |

**⚠️ 关键：日期过滤使用 `published_at` 字段，而非文件日期**

日期过滤逻辑：
1. 优先使用新闻的 `published_at` 字段
2. 若不存在，使用 `fetched_at` 的日期部分
3. 只展示日期范围内**发布**的新闻

## Step 4: L2 深度（摘要与金句）⚠️ 标记已读

**用户输入编号后，必须执行以下操作：**

### 4.1 标记为已读

```bash
# 读取 state.json
state_file="./.creator-space/news_sources/state.json"
state=$(cat "$state_file")

# 更新 read_urls
url_hash="[当前新闻的url_hash]"
read_urls=$(echo "$state" | jq -r '.read_urls')

# 添加已读记录
state=$(echo "$state" | jq --arg hash "$url_hash" \
  --arg url "[新闻URL]" \
  --arg title "[新闻标题]" \
  --arg published "[发布日期]" \
  '.read_urls[$hash] = {
    "url": $url,
    "title": $title,
    "read_at": (now | todate),
    "published_at": $published
  }')

# 保存
echo "$state" > "$state_file"
```

### 4.2 抓取全文并输出

**⚠️ 错误处理流程（重要）：**

**三层 Fallback 机制**：

```
Layer 1: WebFetch 工具
    ↓ 失败/内容无效（≤500字符）
Layer 2: Jina AI Reader (https://r.jina.ai/)
    ↓ 失败
Layer 3: 基于摘要生成解读
```

**详细步骤**：

1. **Layer 1 - WebFetch**
   - 使用 WebFetch 工具抓取文章内容
   - 检查内容是否有效（长度 > 500 字符）
   - 有效 → 继续处理
   - 无效/失败 → 进入 Layer 2

2. **Layer 2 - Jina AI Reader**
   - 使用 Bash 工具执行：
     ```bash
     curl -s -L "https://r.jina.ai/{原文URL}"
     ```
   - Jina AI Reader 会将网页转换为干净的 Markdown 格式
   - 检查返回内容是否有效（长度 > 200 字符且非错误页面）
   - 有效 → 继续处理
   - 无效/失败 → 进入 Layer 3

3. **Layer 3 - 基于摘要生成**
   - 基于新闻标题和摘要生成深度解读
   - 明确告知用户："⚠️ 原文获取受限，基于已有信息生成解读"
   - 提供标题、来源摘要、创作角度

**❌ 禁止做法**：
- 禁止使用 WebSearch 搜索相关内容（可能导致信息偏差）
- 禁止跳转去获取其他新闻
- 禁止自作主张更换目标新闻
