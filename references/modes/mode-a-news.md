# 模式 A：AI新闻推荐模式 (Radar)

**触发条件**：
- 用户问"今天有什么AI新闻"
- 用户说"推荐资讯"、"查看新闻"

## Step 1: 抓取与聚合新闻

```bash
python scripts/fetch_news.py
```

**v1.0 聚合逻辑**：
1. 同时抓取 12 个新闻源（详见 `references/news_sources.md`）
2. 多源识别：合并提及同一核心事件的条目
3. 信息融合：汇总各家数据点，保留所有原始 URL

**输出示例**：

## Step 2: 选题雷达过滤（v4）

不是只按兴趣过滤，而是按“分身创作目标”过滤：

1. 读取 `./.writing-style/system/config.json` 中的兴趣与优先赛道
2. 读取 `./.writing-style/persona/stance_topics.json`（是否符合立场边界）
3. 读取 `./.writing-style/content/metadata/content_map.jsonl`（最近是否重复选题）
4. 读取 `./.writing-style/analytics/reviews.jsonl`（当前最需要补的内容类型）

输出每条新闻的评分：
- 赛道匹配度
- 价值观兼容度
- 新鲜度
- 可转化为内容的难度

## Step 3: 生成推荐列表

为每条新闻生成结构化“选题包”：

```markdown
## 1. GPT-5 发布 / GPT-5 正式发布 ⭐⭐⭐⭐⭐
**摘要**：OpenAI 发布 GPT-5，推理能力提升 3x，支持多模态输入输出。对你的价值：可用于构建更智能的写作助手。
**选题建议**：争议观点、实操教程、行业点评（三选一）
**资产链接**：关联概念 - "大语言模型"、"多模态AI" (`assets/concepts/llm.json`)
**记忆调用**：可复用个人案例（若有）
**来源**：Ben's Bites, TLDR AI
```

## Step 4: 用户交互与创作转化

### 场景 1：查看详情
- 用户输入编号（如 "1"）
- 使用 WebFetch 获取全文并展示

### 场景 2：创作转化（推荐）
1. 获取全文内容
2. 生成 3 个可执行角度（观点型/教程型/案例型）
3. 平台匹配：推荐适合的发布平台（Twitter 短评 / 公众号深度文）
4. **用户选择**：
   - 立即写作 → 跳转**模式 C：平台写作**
   - 暂存灵感 → 跳转**模式 B：资产管理**
