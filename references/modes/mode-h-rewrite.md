# 模式 H：内容改写包装模式 (Content Rewrite) 🆕

**触发条件**：
- 用户说"改写这篇文章"、"把这个包装成我的风格"、"转发到小红书"
- 用户提供别人的文章链接或文本
- 用户想将他人内容改编成符合个人风格的文章

**核心目标**：将他人内容改写成用户风格的文章，同时尊重原创，判断是否需要引用。

**与模式 G 的区别**：
- 模式 G（外部材料学习）：提炼写作技巧 → 更新风格档案
- 模式 H（内容改写包装）：改写文章内容 → 生成新文章

---

## Step 1: 内容获取与类型判断

### 1.1 获取原文

```bash
# 若是链接，使用 WebFetch
WebFetch(url)

# 若是本地文件，使用 Read
Read(file_path)

# 若是直接粘贴的文本
直接读取用户输入
```

### 1.2 内容类型智能判断（关键步骤）

**分析原文，判断属于以下哪一类**：

**类型 A：个人观点类**（需要引用原作者）
- 特征识别：
  - ✓ 包含作者个人经历、案例、故事
  - ✓ 包含作者独特的观点、理论、见解
  - ✓ 大量使用第一人称（"我"、"我的经验"）
  - ✓ 包含作者原创的概念、框架、方法论
- 典型示例：
  - Dan Koe 的一人公司思考
  - 个人成长心得、创业复盘
  - 作者独创的理论体系

**类型 B：通用知识类**（可完全改写）
- 特征识别：
  - ✓ 工具推荐、使用教程、操作指南
  - ✓ 行业新闻、趋势分析、数据报告
  - ✓ 客观事实、科普知识、技术说明
  - ✓ 多人共识的方法论、最佳实践
- 典型示例：
  - AI 工具推荐合集
  - ChatGPT 使用教程
  - 行业趋势分析报告

**类型 C：混合类型**（部分引用 + 部分改写）
- 既包含个人观点，又包含通用知识

**判断输出示例**：
```
📊 内容类型分析完成

原文信息：
- 标题：《如何用一人公司实现财务自由》
- 来源：Dan Koe 个人博客
- 字数：2500字

内容类型：🔴 个人观点类

分析依据：
  ✓ 作者个人创业经历占比：60%
  ✓ 原创理论"价值堆栈"框架：1个
  ✓ 第一人称表达占比：45%
  ✓ 个人案例数量：3个

内容结构拆解：
【核心观点】
1. 个人观点：一人公司的本质是构建个人价值系统 → 🔴 需引用
2. 通用知识：专注比多元化更重要 → ✅ 可改写
3. 个人方法：价值堆栈理论（Dan Koe 原创）→ 🔴 需引用

建议处理方式：混合模式（60% 引用 + 40% 改写）
```

### 1.3 询问用户处理方式

```
📌 根据内容分析，建议使用【混合模式】处理

请选择处理方式：

1. 【引用模式】保留原作者信息 + 加入我的评论解读
   - 适合：个人观点类内容
   - 示例："Dan Koe 在文章中提到... 我的理解是..."
   - 版权风险：✅ 低（明确标注来源）

2. 【改写模式】完全用我的风格重写 + 加入我的思考
   - 适合：通用知识类内容
   - 示例：完全重构内容结构和表达方式
   - 版权风险：⚠️ 中（需确保内容通用性）

3. 【混合模式】个人观点引用 + 通用知识改写
   - 适合：混合类型内容（推荐）
   - 示例：分段处理，原创部分引用，通用部分改写
   - 版权风险：✅ 低（平衡原创性）

请输入选择（1/2/3）：
```

## Step 2: 内容提取与重构

### 2.1 核心信息分层提取

**提取并标记内容来源**：

```
📚 原文内容结构分析

【核心观点】（3个）
1. 一人公司的本质是构建个人价值系统
   - 来源：🔴 作者经验（Dan Koe 创业5年心得）
   - 处理：需引用原作者

2. 专注单一领域比多元化更有效
   - 来源：✅ 普遍共识（多位创业者认同）
   - 处理：可完全改写

3. 利用 AI 工具构建价值堆栈
   - 来源：🟡 混合型（作者方法 + 工具推荐）
   - 处理：方法引用 + 工具改写

【支撑论据】
- 作者案例：Dan Koe 从0到100万订阅的过程 → 🔴 需引用
- 行业数据：80%的一人公司年收入<10万美元 → ✅ 可改写
- 工具推荐：Notion + ChatGPT + Canva 组合 → ✅ 可改写

【金句/原创表达】
- "Build in public, learn in private" → 🔴 需引用（作者原创）
- "专注是唯一的护城河" → ✅ 可改写（通用概念）

---

💡 提取总结：
- 需引用内容：55%
- 可改写内容：45%
- 建议：混合模式 + 加入30%个人解读
```

### 2.2 融入用户个人风格

**读取用户风格档案**：
```bash
# 检查人格状态
profile_initialized=$(cat .writing-style/profile.json | jq -r '.initialized')

if [ "$profile_initialized" == "true" ]; then
    # 读取个人风格特征
    tone=$(jq -r '.expression_preference.tone' .writing-style/profile.json)
    thinking=$(jq -r '.thinking_mode.logic_type' .writing-style/profile.json)
    first_person=$(jq -r '.expression_preference.first_person_usage' .writing-style/profile.json)
else
    # 使用默认去AI化风格
    echo "⚠️ 当前使用默认风格，建议先建立个人风格档案"
fi
```

**风格转换对照表**：

| 原文风格 | 用户风格 | 转换示例 |
|---------|---------|---------|
| 正式专业 | 轻松幽默 | "I spent 5 years..." → "说实话，这5年我踩了无数坑..." |
| 长句陈述 | 短句节奏 | "The key is..." → "核心就一个字：专注。" |
| 抽象概念 | 具体案例 | "Build systems" → "你想想看，我之前..." |

### 2.3 加入个人解读（关键增值）

**基于用户思维方式**，添加个人视角（建议 ≥40%，最低 30%）：

**示例 1：案例驱动思维**
```
【原文观点】
专注单一领域比多元化更有效

【个人解读】
这个我深有体会。去年我同时做了5个项目：
- 写公众号
- 做播客
- 开线上课
- 搞咨询
- 运营社群

结果呢？每个都半死不活，累得要死收入还不如预期。

后来我砍掉4个，专心做公众号 + AI工具分享，3个月粉丝从800涨到2万，
月收入翻了5倍。这就是专注的力量。
```

## Step 3: 生成改写版本（分模式执行）

### 3.1 引用模式（个人观点类）

**文章结构模板**：
```markdown
【标题】（用户风格改写）
原标题：How to Build a One-Person Business
改写后：我从 Dan Koe 的创业思考中学到了什么

【开头】（建立引用语境）
最近读了 Dan Koe 关于一人公司的文章，很有启发。
他用5年时间从0做到百万订阅，文章里提到了几个核心观点。
我结合自己的经验，聊聊我的理解。

【正文】（引用 + 解读交替）
## 1. 一人公司的本质：构建个人价值系统

Dan Koe 说："一人公司的本质是构建一个会随时间复利增长的个人价值系统。"

这句话我琢磨了很久。他的意思是...（个人理解）

我自己的实践是...（个人案例）

【结尾】（总结 + 引用来源）
以上是我读 Dan Koe 文章的一些思考和实践心得，希望对你有帮助。

📌 原文信息：
- 作者：Dan Koe
- 标题：How to Build a One-Person Business
- 链接：[原文链接]
```

**引用规范（必须遵守）**：
- ✅ 明确标注原作者姓名
- ✅ 使用引用标识（"他说"、"作者提到"、引号）
- ✅ 个人解读占比 ≥ 40%
- ✅ 结尾附上原文链接和作者信息
- ✅ 原创金句必须加引号
- ❌ 不得将他人观点说成自己的

### 3.2 改写模式（通用知识类）

**改写要求（必须达到）**：
- ✅ 100%用户风格重写（句式、用词、节奏）
- ✅ 重构内容结构（不照搬原文大纲）
- ✅ 加入个人案例占比 ≥ 30%
- ✅ 加入独特视角/经验/踩坑心得
- ✅ 通过原创度检查（建议 > 70%）
- ❌ 不标注原文来源（因为是通用知识）
- ❌ 避免大段直接翻译原文

### 3.3 混合模式（推荐）

**分段处理策略**：

```
【内容分段标记】
段落1：作者创业经历（个人观点）→ 引用模式
段落2：一人公司定义（通用概念）→ 改写模式
段落3：价值堆栈理论（原创方法）→ 引用模式
段落4：工具推荐（通用知识）→ 改写模式
段落5：作者总结（个人观点）→ 引用模式 + 个人解读
```

## Step 4: 平台适配 + 质量检查

### 4.1 选择目标平台

询问用户选择发布平台（小红书/Twitter/微信公众号），执行对应平台规则（详见 mode-c-platform.md）

### 4.2 五遍检查机制（新增版权核查）

**Pass 1 - 初稿生成**

**Pass 2 - 版权核查**（🆕 内容改写专属）
```
🔍 版权检查报告

【引用规范检查】
✅ 原作者信息已标注：Dan Koe
✅ 引用标识使用正确："他说"、引号（共5处）
✅ 原文链接已附上
✅ 原创金句已加引号："Build in public, learn in private"

【内容占比分析】
- 引用原文内容：55%
- 个人解读/案例：45%
- 建议：✅ 合格（个人解读 ≥ 40%）

【风险评估】
风险等级：✅ 低风险
原因：
  - 明确标注来源
  - 充分个人解读
  - 遵守引用规范

💡 优化建议：
- 可以再增加1个个人案例，提升原创度到50%
```

**Pass 3 - 风格克隆**（详见 quality-control.md）

**Pass 4 - 平台规范检查**

**Pass 5 - 用户最终审查**

**Pass 6 - 原创与分身一致性校验（v4新增）**
- 检查是否满足个人解读占比阈值
- 检查是否符合 `persona/do_dont_say.json`
- 检查是否出现与 `memory/beliefs.json` 冲突的核心立场
