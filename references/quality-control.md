# 质量控制：四遍检查机制

平台写作完成后，必须执行四遍检查以确保质量。

## Pass 1 - 策略对标与素材注入 (Blueprint & Injection)

在动笔前，必须明确本次创作的“骨架”与“血肉”：
1. **策略对标**：检索 `assets/strategies/` 或 `learned_techniques`，声明本次套用的爆款框架（如：dontbesilent 框架）。
2. **侦察痛点**：若为选题写作，必须列出 WebSearch 挖掘到的全网 3 大痛点或误区。
3. **资产注入**：强制从库中挑选至少 1 条金句、1 个实战案例或 1 个核心概念，拒绝空洞生成。

## Pass 2 - 事实核查与版权评估 (Fact & Ethics Check)

**1. 识别关键信息点**：
- 数据（统计数字、百分比）
- 引用（专家观点、研究结论）
- 产品名称（工具、公司、技术）

**2. [新增] 伦理与原创度核查**：
- **查重预估**：若为模式 H 改写，评估与原文的差异，确保“个人解读”比例 ≥ 40%。
- **版权声明**：检查是否按照“致敬转发”或“去痕重塑”要求处理了出处。

**3. 验证流程**：
```bash
# 使用 WebSearch 验证每个关键信息点
WebSearch("GPT-5 发布日期")
WebSearch("OpenAI 官方公告")
```

**修正错误**：
- 更新不准确的数据
- 补充缺失的来源链接
- 纠正拼写错误

## Pass 3 - 风格克隆 (自适应人格模式 + 强制展示修改过程)

**检查风格成熟度**：
```bash
# 读取风格数据
profile_initialized=$(cat ./.creator-space/system/profile.json | jq -r '.initialized')
articles_analyzed=$(cat ./.creator-space/system/profile.json | jq -r '.statistics.articles_analyzed // 0')
do_dont_say_count=$(cat ./.creator-space/persona/do_dont_say.json | jq -r '[.do // [], .dont // []] | add | length')
voice_style_tone=$(cat ./.creator-space/persona/voice_style.json | jq -r '.tone // "default"')
```

**风格成熟度判断**：
| 条件 | 判断结果 |
|-----|---------|
| `initialized=false` | **未初始化** → 使用默认风格 |
| `articles_analyzed < 3` | **风格未成熟** → 混合模式 |
| `do_dont_say_count < 5` | **风格未成熟** → 混合模式 |
| `voice_style_tone="default"` | **风格未成熟** → 混合模式 |
| 以上都不满足 | **风格成熟** → 个人风格优先 |

### AI痕迹硬伤清单（所有模式都检查）

无论风格成熟度如何，以下问题始终标记：

| 硬伤类型 | 示例 | 处理方式 |
|---------|------|---------|
| 模糊归因 | "专家认为"、"研究表明" | 未成熟→强制修改；成熟→提醒 |
| 宣传性语言 | "令人叹为观止"、"充满活力的" | 未成熟→强制修改；成熟→提醒 |
| 空洞开头 | "希望这对你有帮助"、"当然！" | 所有模式→强制删除 |
| 否定式排比 | "这不仅仅是X，而是Y" | 未成熟→强制修改；成熟→提醒 |
| 三段式列举 | "包括A、B和C机会" | 所有模式→建议变化 |
