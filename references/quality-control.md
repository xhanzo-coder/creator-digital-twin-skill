# 质量控制：四遍检查机制

平台写作完成后，必须执行四遍检查以确保质量。

## Pass 1 - 初稿生成

完成第一版内容，包含所有核心信息。

## Pass 2 - 事实核查

**识别关键信息点**：
- 数据（统计数字、百分比）
- 引用（专家观点、研究结论）
- 产品名称（工具、公司、技术）

**验证流程**：
```bash
# 使用 WebSearch 验证每个关键信息点
WebSearch("GPT-5 发布日期")
WebSearch("OpenAI 官方公告")
```

**修正错误**：
- 更新不准确的数据
- 补充缺失的来源链接
- 纠正拼写错误

---

## Pass 3 - 风格克隆 (自适应人格模式 + 强制展示修改过程)

**检查人格状态**：
```bash
# 读取人格状态
profile_initialized=$(cat ./.writing-style/system/profile.json | jq -r '.initialized')
```

### 情况 1：已初始化个人风格 (`initialized: true`)

**执行步骤**：

1. **读取完整风格档案**：
   - 用词习惯（高频词、避免词、口语化程度）
   - 句式偏好（平均句长、长短句比例）
   - 表达习惯（第一人称使用、比喻使用）
   - 实时学习积累的修改偏好（`learned_from_edits`）

2. **逐句对照检查**：
   - 识别不符合用户风格的句子
   - 对每个需要修改的地方进行改写

3. **🔴 强制展示修改过程**（原始需求核心要求）：

**输出格式**：
```
📝 Pass 3 - 风格克隆检查报告

修改 1/5:
原文：这项技术在行业中具有至关重要的意义。
改为：这个技术在行业里挺关键的。
修改原因：
  - 你不会用"这项"，习惯用"这个"（更口语化）
  - 你避免使用"至关重要"（AI词汇），更喜欢"挺+形容词"
  - 你不会用"具有...意义"，会直接说"挺关键的"

修改 2/5:
原文：此外，它还能提升工作效率。同时，也降低了成本。
改为：一方面提升效率，另一方面降低成本。
修改原因：
  - 你避免使用"此外"、"同时"（从你的修改记录中学到）
  - 你喜欢把连续短句合并为并列句
  - 你习惯用"一方面...另一方面"的逻辑连接

修改 3/5:
原文：让我们深入探讨一下这个问题的本质。
改为：你想想看，这事儿本质上是什么？
修改原因：
  - 你不会用"让我们深入探讨"（过于正式）
  - 你喜欢用"你想想看"引导读者思考（高频词）
  - 你习惯用疑问句而不是陈述句

---
✅ 共修改 5 处，已全部符合你的个人风格
```

### 情况 2：使用默认人格 (`initialized: false`)

- 读取 `references/default-humanizer-style.md`
- 执行去AI化处理（24条规则）
- 同样展示修改过程，但原因改为"去AI化规则"：

```
修改 1/3:
原文：此外，该技术展示了持久的影响力。
改为：这个技术影响深远。
修改原因：
  - 删除AI词汇："此外"、"展示"、"持久的"
  - 简化表达，避免宣传性语言
```

## Pass 4 - 平台规范检查

**对照 `platform_rules/*.json` 检查所有预设项**：

### 小红书规范检查清单

- [ ] 字数：800-1200 字
- [ ] 标题：包含数字或情绪词
- [ ] Emoji：每 50-80 字一个
- [ ] 话题标签：3-5 个
- [ ] 段落：单段不超过 4 行
- [ ] 开头：前 3 句必须有 Hook

### Twitter 规范检查清单

- [ ] 每条推文 ≤ 280 字符
- [ ] 第一条包含 Hook
- [ ] Thread 总数 ≤ 10 条
- [ ] 每条推文独立完整

### 微信公众号规范检查清单

- [ ] 字数：3000-5000 字
- [ ] 小标题：每 500-800 字一个
- [ ] 引言：100-200 字
- [ ] 总结：200-300 字
- [ ] 排版：段落间距 1.5-2 倍

**若不符合规范**：
- 自动调整格式
- 展示调整内容
- 询问用户确认
